隐私政策 — ELA

最后更新：2026 年 4 月 24 日  ·  English

ELA（以下简称"本扩展"）是一款 Chrome 扩展，通过文字转语音、AI 翻译和查词功能，帮助你更轻松地阅读外语内容——所有操作均在侧边栏中完成。本政策说明扩展会访问哪些数据、如何使用，以及数据何时（如有）会离开你的设备。

1. 我们存储的数据

ELA 使用 Chrome 内置的 chrome.storage.local API 将以下数据仅存储在你的本地设备上，不会发送到 ELA 运营的任何服务器。

类别示例
AI 服务设置你的 OpenAI API Key、所选 TTS 语音模型
自定义按钮配置你在设置页面定义的 AI 按钮名称和 Prompt 文本

ELA 不收集、存储或传输任何个人身份信息（如姓名、邮箱、地理位置、浏览历史等）。

2. 数据的使用方式

本地存储的数据仅用于：

  • 代表你向 OpenAI API 发起认证请求（API Key）
  • 确定生成语音时使用的声音模型（语音模型偏好）
  • 显示并执行你配置的 AI 功能（自定义按钮 Prompt）

3. 发送给第三方服务的数据

ELA 的核心功能——文字转语音、翻译和查词——均由 OpenAI API 提供支持。当你使用这些功能时，以下数据将直接从你的浏览器发送至 OpenAI：

功能发送给 OpenAI 的数据
文字转语音（播放）你在网页上选中的文字
AI 按钮（如翻译、查词、自定义功能）选中的文字 + 该按钮对应的 Prompt

上述数据直接从你的浏览器传输至 OpenAI，不经过 ELA 的任何服务器。你的 API Key 仅用于授权这些请求，开发者无法获取。

请查阅 OpenAI 的隐私政策，了解其数据处理方式：
https://openai.com/policies/privacy-policy

如果你不主动使用扩展功能，任何数据都不会被发送。ELA 不会在后台运行，也不会监控你的浏览行为。

4. 数据存储与保留

所有设置（API Key、语音模型、自定义 Prompt）均通过 chrome.storage.local 存储在本地，直到你：

  • 在设置页面手动清除
  • 卸载扩展（卸载后所有存储数据将被自动删除）

ELA 的开发者无法访问你的本地数据，也无法代表你检索或删除这些数据。

5. 权限说明

权限用途
storage将 API Key、语音模型偏好和自定义按钮 Prompt 保存在本地设备上
sidePanel显示 ELA 侧边栏，用于展示选中文字、播放控件和 AI 结果
内容脚本（<all_urls>）检测你在任意网页上选中的文字，并将其传递到侧边栏。由于外语内容分布在数不清的网站上，因此需要广泛的主机权限。该脚本仅在你打开侧边栏且启用开关后才会激活，不会在其他情况下读取页面内容。

6. 数据共享

ELA 不会：

  • 向任何第三方出售、出租或共享你的数据
  • 将你的数据传输至 ELA 或其开发者运营的任何服务器
  • 将你的数据用于广告或数据分析
  • 访问或存储你所访问网页的内容（仅限你主动选中的文字）

7. 未成年人隐私

ELA 是面向所有年龄段用户的通用语言学习工具，不收集任何用户（包括未成年人）的个人信息。
